3D Asset Development



No matter if you are a company wanting to showcase a product, and artist making an installation, a designer setting up an interactive visualization, or a game developer setting up game levels one key aspect of the experience is creating detailed 3D environment. Depending on the style, scale and application not only do elements need to match the overall mood of the experience, but the elements have to be optimized, varied and sometimes interactive in order to fulfill the vision of the project. Further consideration is to be made on the side of User Interface (UI) where default solutions are not acceptable for the custom User Experience (UX), have to encode sometimes very complex sequence of events in interaction, and yet have to comply with a rigid set of supported types of elements or events.

When creating virtual environments one of the biggest complications is finding or creating assets that would work well with the engine in use while looking the part in the context. To complicate matters more, different mediums have different capacities to not only process but to display various levels of detail and different features for rendering, physics or interaction. Will there be a range of types to be manually distributed or do they have to be generated dynamically? Are there custom effects or is it a static element? Would an element have to trigger an event or have a predefined animation played on a certain condition? Will it be experienced in a particular place or follow the user through the environment?

To address these Polytope can provide individual and custom development of assets for your virtual environments; extend assets to work for a given usage; augment an asset to work with scripting or interaction system; set up the placeholder sequence of elements and events necessary to test the experience; or some combination of the above. Be that a conventional 3D assets using Blender or Rhinoceros 3D, or easily customizable elements using Grasshopper 3D or UI elements using Adobe Suite - they all ultimately have to come together in a cohesive experience (in our case primarily wotksing with Unity3D).

Desarollo de Assets 3D



No importa si son una empresa que desea mostrar un producto, o un artista que realiza una instalación, o un diseñador que configura una visualización interactiva, o un desarrollador de juegos que configura niveles de juego - un aspecto clave de la experiencia es la creación de un entorno 3D detallado. Dependiendo del estilo, escala y aplicación, no solo los elementos deben coincidir con el estado de ánimo general de la experiencia, sino que los elementos deben ser optimizados, variados y, a veces, interactivos para cumplir con la visión del proyecto. Además, se debe tener consideración en el lado de la Interfaz de Usuario (UI) donde las soluciones predeterminadas no son aceptables para la Experiencia de Usuario (UX) personalizada, tienen que codificar a veces de manera muy compleja secuencia de eventos en interacción, y, sin embargo, debe cumplir con un conjunto rígido de tipos de elementos o eventos admitidos.

Al crear entornos virtuales, una de las mayores complicaciones es encontrar o crear assets que funcionen bien con el motor en uso mientras parece a la altura en el contexto. Para complicar más las cosas, los diferentes medios tienen diferentes capacidades no solo para procesar, sino también para mostrar varios niveles de detalle y diferentes características de renderizado, física o interacción. ¿Habrá una variedad de tipos para distribuir manualmente o tendrán que generarse dinámicamente? ¿Hay efectos personalizados o es un elemento estático? ¿Un elemento tendría que causar un evento o tener una animación predefinida reproducida en una condición? ¿Se experimentará en un lugar en particular o seguirá al usuario a través del entorno?

Para abordar estos Polytope puede proporcionar desarrollo de activos individual y personalizado para sus entornos virtuales; extender los activos para que funcionen para un uso determinado; aumentar un activo para trabajar con codificación o sistema de interacción; configurar la prototipo de secuencia de elementos y eventos necesarios para probar la experiencia; o alguna combinación de los anteriores. Sea unos assets 3D convencionales usando Blender o Rhinoceros 3D, o bien elementos fácilmente personalizables usando Grasshopper 3D o Elementos de la Interfaz de Usuario con Adobe Suite: en cualquier caso, todos deben unirse en una experiencia coherente (en nuestro caso trabajamos con Unity3D).

Desenvolupament d'Assets 3D

Per complicar més les coses, els diferents mitjans tenen diferents capacitats no només per processar, sinó també per mostrar diversos nivells de detall i diferents característiques de renderitzat, física o interacció. Sigui uns assets 3D convencionals fent servir Blender o Rhinoceros 3D, o bé elements fàcilment personalitzables fent servir Grasshopper 3D o elements de la Interfície d'Usuari amb Adobe Suite: en qualsevol cas, tots han d'unir en una experiència coherent (en el nostre cas treballem amb Unity3D).
